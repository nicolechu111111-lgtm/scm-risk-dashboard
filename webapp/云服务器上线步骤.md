# SCM 看板云服务器上线步骤

## 推荐服务器

买一台最小规格 Linux 云服务器即可：

- 系统：Ubuntu 22.04 或 Ubuntu 24.04
- 配置：1 核 / 1GB 内存起步，2GB 更稳
- 硬盘：20GB 以上
- 开放端口：`8080`

如果后续要绑定域名和 HTTPS，再开放 `80` 和 `443`。

## 第一次上线

登录服务器后安装 Docker：

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable --now docker
```

把本项目上传到服务器，比如放在：

```text
/opt/scm-dashboard
```

进入项目目录：

```bash
cd /opt/scm-dashboard
```

设置共享密码：

```bash
cat > .env <<'EOF'
SCM_DASHBOARD_PASSWORD=这里改成你的共享密码
EOF
```

启动看板：

```bash
sudo docker compose up -d --build
```

访问：

```text
http://服务器IP:8080/dashboard
```

第一次打开后，输入共享密码，然后上传最新版 Follow Up。

## 日常使用

- 你上传新的 Follow Up 后，系统会重新计算。
- 其他人刷新网页后看到同一版最新看板。
- 上传过的 Follow Up 和生成结果保存在 Docker volume：`scm-dashboard-data`。
- 服务器重启后服务会自动恢复。

## 更新代码

如果后续我继续帮你改看板逻辑，把新代码上传到服务器后运行：

```bash
cd /opt/scm-dashboard
sudo docker compose up -d --build
```

## 重要提醒

- 不要把共享密码设置得太简单。
- 如果直接用 IP + 8080 访问，适合先内部试用。
- 正式长期使用建议绑定域名并加 HTTPS。
